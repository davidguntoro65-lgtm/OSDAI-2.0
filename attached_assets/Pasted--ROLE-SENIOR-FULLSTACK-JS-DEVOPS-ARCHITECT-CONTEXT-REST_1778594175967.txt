# CONTEXT: RESTRUCTURING FOR CPANEL DEPLOYMENT (PHUSION PASSENGER)

Please restructure this project to fix critical deployment issues found in production. The app must run on a cPanel Shared Hosting environment using Node.js and PostgreSQL.

### 1. ARCHITECTURE REQUIREMENTS (PRECISION)
- **Root-Level `app.js`**: Create a robust entry point for Phusion Passenger. It must:
  - Initialize Express or the current backend framework.
  - Bridge the API routes (e.g., `/api`).
  - Serve static files from the `/dist` directory.
  - Handle SPA routing (fallback to `dist/index.html`).
- **Prisma Configuration**: Ensure `schema.prisma` uses `provider = "postgresql"` and `url = env("DATABASE_URL")`.
- **Package.json**: Update scripts to include:
  - `"build": "vite build"`
  - `"start": "node app.js"`
  - Ensure all production dependencies are in "dependencies", not "devDependencies".

### 2. FIXING CRITICAL BUGS IDENTIFIED IN AUDIT
- **MIME Type Error**: Ensure the production `index.html` (inside `dist`) DOES NOT reference `.tsx` files. It must only reference compiled `.js` assets.
- **API Endpoint Logic**: In `app.js` and frontend services, ensure the API URL is dynamic. Use relative paths `/api/...` or detect `location.hostname` to avoid hardcoded `localhost:3000`.
- **Environment Handling**: The app must be able to read environment variables from both `.env` and `process.env` (for cPanel runtime).
